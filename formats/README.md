# Formats
## Deliverable Format Catalog

---

## What formats are

Format files are deliverable-specific instantiations of the Harness's universal layered deliverable governance (HARNESS.md §19). Each file defines the layer model, governance rules, and AI behavior for a specific type of deliverable.

Formats are not Harness operations — those live in `skills/`. They are methodology files: how to structure and govern a specific class of work.

---

## When to load a format

Load the relevant format file when a project is producing a structured deliverable. Read it alongside `HARNESS.md`, not instead of it. The universal principles in §19 govern; the format file provides the mechanics.

---

## Format Catalog

| Format | File | Use for |
|---|---|---|
| Layered Deck | `layered-deck.md` | Decks, reports, proposals, frameworks — any structured knowledge artifact governed by an argument layer |
| Product Design | `product-design.md` | Interface, flow, and mockup work — any initiative producing a design artifact (wireframe, HTML mockup, Figma screens). Scoped to the design phase; does not cover production engineering. |
